# Plan 4: Web UI

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a React SPA that provides chat, document management, and system dashboard -- served as static files by the existing Hono server.

**Architecture:** `@opendocuments/web` is a Vite + React + TypeScript + Tailwind CSS app. Built output is served by `@opendocuments/server` as static files at `/`. API calls go to `/api/v1/*`. Chat uses SSE streaming via `POST /api/v1/chat/stream`.

**Tech Stack:** React 19, Vite 6, TypeScript, Tailwind CSS 4, Zustand (state), react-markdown + rehype (rendering), Shiki (code highlighting)

**Spec Reference:** `docs/superpowers/specs/2026-03-26-opendocuments-design.md` section 6.3-6.4

---

## Scope

**In scope:**
- Chat page (main) with SSE streaming, source citations, profile toggle
- Document management page (list, upload, delete)
- Settings page (config view, RAG profile)
- System health page (doctor output)
- Dark/light theme
- Responsive sidebar layout
- API client module

**Out of scope (Phase 2+):**
- Connectors page, Plugins page, Workspaces page
- Command palette (Cmd+K), keyboard shortcuts
- i18n, conversation sharing, embeddable widget
- Auth/login page
